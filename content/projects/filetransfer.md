# fileTransfer

A simple C-based TCP file transfer utility supporting both sending and receiving files, using a custom transfer protocol.

## Features

- Transfer files over the network using raw TCP sockets.
- Operates in either `send` (client) or `recv` (server) mode.
- Modular codebase: separate code for sending, receiving, and protocol utilities.
- Provides transfer status updates and robust error handling.

## Directory Structure

- `cloud.c` – Main entry point, dispatches to send/recv functionality.
- `send/` – Handles the client (sending) side.
- `recv/` – Handles the server (receiving) side.
- `shared/` – Common utilities (error handling, protocol).

## Usage

### Compile

Make sure you have GCC installed. To compile:

```sh
gcc cloud.c send/cloudSend.c send/setupTCPClient.c send/printSFileStatus.c recv/cloudRecv.c recv/setupTCPServer.c recv/HandleTCPClient.c recv/AcceptTCPConnection.c recv/printRFileStatus.c shared/DieWithUserMessage.c shared/Shared.h shared/goAhead.c shared/printSocketAddress.c shared/waitForGo.c -o cloud
```

(You may need to adjust the file list depending on any additional implementation files and header dependencies.)

### Running as Sender

```sh
./cloud send <host> <port> <file1> [file2 ...]
```

- `<host>`: IP or hostname of the receiver.
- `<port>`: Port number where the receiver is listening.
- `<fileN>`: Path(s) to file(s) to transfer.

Example:
```sh
./cloud send 192.168.1.5 8080 ./example.txt
```

### Running as Receiver

```sh
./cloud recv <port> <output_directory>
```

- `<port>`: Port number to listen on.
- `<output_directory>`: Where to store received files.

Example:
```sh
./cloud recv 8080 ./received_files/
```

## Custom Protocol Details

- The sender first sends the file name, size, and a separator string.
- The sender waits for a "go ahead" signal from the receiver before streaming the file content.
- Each transfer is tracked and status is displayed in the terminal.

## Dependencies

- Standard C library.
- POSIX sockets (`sys/socket.h`, etc).
- Linux/Unix–like OS.
